Installation

XUZU requires Python ≥ 3.9 and PyTorch ≥ 2.0. Install dependencies and the package in editable mode.

bash
# 1. Create a virtual environment (recommended)
python -m venv xuzu-env
source xuzu-env/bin/activate   # Windows: xuzu-env\Scripts\activate

# 2. Install core dependencies
pip install torch>=2.0.0 numpy>=2.1.0

# 3. Install XUZU in editable mode
cd XUZU_PROJECT
pip install -e .

# 4. Verify
python -c "import xuzu; print(xuzu.__version__, '|', xuzu.__author__, '@', xuzu.__lab__)"
# → 1.0.0 | Hamza A @ terminalBio

# Optional: for RNAfold structural eval
pip install RNA>=2.6.0
💡

GPU recommended. Training on CPU is possible but slow. Use --device cuda if an NVIDIA GPU is available, or --device mps on Apple Silicon.

03

Data Format

XUZU trains on JSONL files — one JSON record per line. Each record can include a nucleotide sequence, optional secondary structure, optional binding pocket, and optional measured Kd.

jsonl — aptamers.jsonl
// Minimal record — sequence only
{"seq": "GCGAUAGCUAGCUAGCUAGC"}

// Full record — all fields
{
  "seq":       "GCGAUAGCUAGCUAGCUAGCUA",
  "structure": "(((((.......)))))",
  "pocket":    "SMKDYSFLTQFPGFVKHFNSLGG",
  "kd_nm":     12.5
}
FieldRequiredTypeDescription
seqrequiredstringAptamer nucleotide sequence — DNA or RNA. Modified bases: F, M, L, P.
structureoptionalstringDot-bracket secondary structure. Same length as seq. Defaults to all dots if omitted.
pocketoptionalstringAmino-acid sequence of target binding pocket residues. Standard 20 AA letters.
kd_nmoptionalfloatMeasured binding affinity Kd in nanomolar. Required only for GER loop training.
ℹ

Data sources for KPC-2 aptamers: AptaBase, SELEX-DB, literature-curated sequences from PRISMA-reviewed papers (2011–2026). Extract KPC-2 pocket residues from PDB 3RXW — active site: Ser70, Lys73, Ser130, Asn132, Glu166.

Generating Training Data from Scratch

python — generate_sample_data.py
import json, random

# KPC-2 beta-lactamase active site pocket (from PDB 3RXW)
KPC2_POCKET = "SMKDYSFLTQFPGFVKHFNSLGGDGVQVAKNTSMAQGLGSG"

# Generate random RNA aptamers as training scaffold (replace with real seqs)
def random_rna(length=40):
    return ''.join(random.choice('AUGC') for _ in range(length))

records = []
for _ in range(500):
    seq = random_rna(random.randint(25, 55))
    records.append({
        "seq":     seq,
        "structure": '.' * len(seq),   # use RNAfold for real structures
        "pocket":  KPC2_POCKET,
    })

with open("kpc2_aptamers.jsonl", "w") as f:
    for r in records:
        f.write(json.dumps(r) + "\n")

print(f"Wrote {len(records)} training records")

XUZUConfig — All Parameters

All model and training hyperparameters are controlled via XUZUConfig. Every parameter has a sensible default for aptamer design tasks.

ParameterDefaultDescription
d_model256Hidden dimension for all towers and decoder. Scale to 512 for larger model (~30M params).
n_heads8Attention heads in FEBI blocks. Must divide d_model evenly.
nle_layers6Depth of Nucleotide Language Encoder (Tower 1, FEBI blocks).
sge_layers3Depth of Structure Graph Encoder (Tower 2, RJ blocks).
sge_heads4Graph attention heads in RJ blocks.
tpe_layers3Depth of Target Protein Encoder (Tower 3, FEBI blocks).
dec_layers4Depth of diffusion decoder (FEBI blocks).
diffusion_T100Number of diffusion timesteps. Higher = smoother denoising, slower training.
max_seq_len128Maximum nucleotide sequence length (including [BOS] and [EOS]).
max_poc_len64Maximum pocket amino-acid sequence length.
dropout0.1Dropout rate in all FEBI and RJ blocks.
lr1e-4Peak AdamW learning rate. Warmed up over warmup_steps, then cosine-decayed.
weight_decay0.01AdamW weight decay. Controls overfitting.
batch_size32Training batch size. Reduce if GPU OOM.
warmup_steps500LR warmup steps before cosine decay begins.
python — custom config
from xuzu import XUZU, XUZUConfig

# Larger model (~30M params) for bigger datasets
cfg = XUZUConfig(
    d_model      = 512,
    n_heads      = 16,
    nle_layers   = 8,
    sge_layers   = 4,
    tpe_layers   = 4,
    dec_layers   = 6,
    diffusion_T  = 200,
    lr           = 5e-5,
    batch_size   = 16,
    warmup_steps = 1000,
)

model = XUZU(cfg)
print(f"Parameters: {model.num_parameters():,}")   # ~30M

Advanced Training (Python API)

Use the Python API for full control over training — custom data pipelines, callbacks, multi-GPU, and logging integrations.

python — full training script
from xuzu import (
    XUZU, XUZUConfig, XUZUTrainer,
    NucleotideTokenizer, build_dataloaders
)

# ── 1. Configure ─────────────────────────────────────────────
cfg = XUZUConfig(
    d_model    = 256,
    nle_layers = 6,
    lr         = 1e-4,
    batch_size = 32,
)

# ── 2. Build model ────────────────────────────────────────────
model = XUZU(cfg)
print(f"XUZU v{xuzu.__version__} | {model.num_parameters():,} params")
print(f"Author: {xuzu.__author__} @ {xuzu.__lab__}")

# ── 3. Load data ──────────────────────────────────────────────
tokenizer = NucleotideTokenizer()
train_loader, val_loader = build_dataloaders(
    path        = "kpc2_aptamers.jsonl",
    tokenizer   = tokenizer,
    val_frac    = 0.1,
    batch_size  = cfg.batch_size,
    max_seq_len = cfg.max_seq_len,
    max_poc_len = cfg.max_poc_len,
    num_workers = 4,
)
print(f"Train: {len(train_loader.dataset)} | Val: {len(val_loader.dataset)}")

# ── 4. Train ──────────────────────────────────────────────────
trainer = XUZUTrainer(
    model    = model,
    device   = "cuda",      # or "cpu" / "mps"
    use_amp  = True,         # mixed precision (GPU only)
    log_file = "train.log",
)

history = trainer.train(
    train_loader = train_loader,
    val_loader   = val_loader,
    epochs       = 150,
    patience     = 15,
    save_path    = "xuzu_kpc2_best.pt",
    eval_every   = 1,
)

# ── 5. Plot training curve ────────────────────────────────────
import json
with open("history.json", "w") as f:
    json.dump(history, f)
⚠

Mixed precision (AMP) is only supported on CUDA devices. Setting use_amp=True on CPU or MPS is silently ignored and training continues in full precision.

07

GER Loop — Closed-Loop Refinement

After base training, the Generation–Evaluation–Refinement (GER) loop fine-tunes the decoder using a binding affinity reward signal. This pushes XUZU to generate aptamers with lower predicted Kd values.

1
Generate
Decoder proposes candidate aptamer sequence from fused multi-modal context.
2
Evaluate
BindingAffinitySurrogate predicts log(Kd) from mean-pooled context embedding.
3
Normalise Reward
Reward = −log(Kd), z-score normalised within batch to reduce gradient variance.
4
Refine
REINFORCE policy gradient updates decoder weights. Entropy bonus maintains sequence diversity.
5
Iterate
Repeat for N refinement steps. Decoder progressively generates better binders.
python — GER loop
from xuzu import XUZU, BindingAffinitySurrogate, GERLoop
import torch

# Load pretrained XUZU base model
model = XUZU.load("xuzu_kpc2_best.pt", device="cuda")

# Initialise surrogate reward model
# (train surrogate first on known Kd values, or use as-is for exploratory refinement)
surrogate = BindingAffinitySurrogate(d_model=model.cfg.d_model).to("cuda")

# Initialise GER loop
ger = GERLoop(
    model       = model,
    surrogate   = surrogate,
    device      = "cuda",
    lr          = 3e-5,
    entropy_coef = 0.01,  # higher → more diverse outputs
)

# Run refinement on a batch
KPC2_POCKET = "SMKDYSFLTQFPGFVKHFNSLGGDGVQ"
poc_ids, poc_mask = model.tpe.tokenize(KPC2_POCKET, "cuda")

for step in range(500):
    # Build a batch of random seed sequences
    B = 16
    seq_ids  = torch.randint(5, 14, (B, 45), device="cuda")
    seq_mask = torch.ones(B, 45, dtype=torch.long, device="cuda")
    adj      = torch.eye(45).unsqueeze(0).expand(B, -1, -1).to("cuda")
    poc_ids_b  = poc_ids.expand(B, -1)
    poc_mask_b = poc_mask.expand(B, -1)

    # Get multi-modal context
    context = model.encode(seq_ids, seq_mask, adj, poc_ids_b, poc_mask_b)

    # GER refinement step
    metrics = ger.refine_step(context, seq_ids, seq_mask)

    if step % 50 == 0:
        print(f"Step {step:>4d} | pg_loss={metrics['pg_loss']:.4f} | "
              f"reward={metrics['mean_reward']:.4f} | H={metrics['entropy']:.4f}")

# Save GER-refined model
model.save("xuzu_kpc2_ger.pt")

Designing Aptamers

bash — CLI design
# Design 50 RNA aptamers targeting KPC-2
python design.py   --pocket   "SMKDYSFLTQFPGFVKHFNSLGGDGVQVAKNTSMAQGLGSG"   --model    xuzu_kpc2_ger.pt   --len      45   --n        50   --temp     0.85   --rna   --evaluate

# With structural constraint (stem-loop motif)
python design.py   --pocket    "SMKDYSFLTQFPGFVK"   --structure "((((((........))))))"   --model     xuzu_kpc2_ger.pt   --len       40 --n 20 --rna
python — programmatic design
from xuzu import XUZU, evaluate_batch

model = XUZU.load("xuzu_kpc2_ger.pt", device="cuda")

aptamers = model.design(
    pocket_seq   = "SMKDYSFLTQFPGFVKHFNSLGGDGVQ",
    seq_len      = 45,
    temperature  = 0.85,   # lower = more conservative
    top_k        = 10,     # nucleus filtering (0 = off)
    n_candidates = 100,
    as_rna       = True,
)

# Evaluate ensemble
metrics = evaluate_batch(aptamers, reference=[])
print(metrics)
# {'n_generated': 100, 'gc_mean': 0.51, 'novelty': 0.87, 'diversity': 0.73, ...}

# Save to FASTA for downstream docking
with open("kpc2_aptamers_xuzu.fasta", "w") as f:
    for i, apt in enumerate(aptamers, 1):
        f.write(f">XUZU_KPC2_{i:03d}\n{apt}\n")

Full KPC-2 Workflow

End-to-end pipeline for designing RNA aptamers targeting KPC-2 β-lactamase for AMR research.

1
Extract KPC-2 Pocket
Download PDB 3RXW. Extract active site residues: Ser70, Lys73, Ser130, Asn132, Glu166, Asn170. Use PyMOL or BioPython to extract pocket AA sequence.
2
Prepare Training Data
Collect aptamer sequences from PRISMA-reviewed literature (2011–2026). Predict structures with RNAfold. Format as JSONL.
3
Train XUZU Base Model
python train.py --data kpc2.jsonl --epochs 200 --device cuda --save xuzu_kpc2.pt
4
GER Refinement
Run 500+ GER steps with KPC-2 pocket as target. Saves GER-refined weights.
5
Generate Candidates
python design.py --pocket "..." --model xuzu_kpc2_ger.pt --len 45 --n 100 --rna --evaluate
6
Validate with HADDOCK 2.4
Dock top candidates against KPC-2 structure. Score with HADDOCK energy. Filter top 10 by binding score.
7
MD Validation
Run 100ns GROMACS MD simulations on top docked complexes. Confirm binding stability.
🔬

terminalBio recommendation: Select top candidates by combining XUZU novelty score > 0.6, GC content 45–55%, negative MFE from RNAfold, and HADDOCK score < −100. This multi-criteria filter gives the highest hit rate for wet-lab validation.
